# 要件整理テンプレート

PowerPoint 資料を生成する前に、このテンプレートを使ってユーザーから必要な情報を引き出します。
**不明な項目はユーザーに確認すること。** 推測で進めない。

---

## 必須項目（必ず確認する）

| 項目 | 質問例 | 記入欄 |
|------|--------|--------|
| **目的・ゴール** | 「この資料で何を達成したいですか？」 | |
| **対象読者** | 「誰に見せる資料ですか？（役職・技術レベル）」 | |
| **スライド枚数** | 「何枚くらいを想定していますか？」 | |
| **出力パス** | 「保存先のファイル名を教えてください（`output/` に保存します）」 | |

---

## 推奨確認項目（不明な場合はデフォルト値を使う）

| 項目 | 質問例 | デフォルト |
|------|--------|-----------|
| **トーン** | 「フォーマル/カジュアルどちらですか？」 | フォーマル |
| **テンプレート** | 「既存の .pptx テンプレートはありますか？」 | `template/` を確認（あれば使用） |
| **言語** | 「スライドの言語は？」 | 日本語 |
| **カラースキーム** | 「ブランドカラーや希望の色はありますか？」 | Modern Blue |
| **コンテンツ素材** | 「テキスト・データ・画像の素材はありますか？」 | なし |

---

## コンテンツ構成の確認

ユーザーから渡された情報をもとに、以下のアウトラインをドラフトしてユーザーに確認を取る。

### アウトライン例（事業提案 8 枚）

```
1. タイトルスライド
   - タイトル、サブタイトル、発表者名、日付

2. 課題・背景
   - 現状の問題点（3 点以内）

3. 解決策の概要
   - 提案するソリューション

4. 詳細説明（1〜2 枚）
   - 機能・特徴・仕組み

5. 期待効果・KPI
   - 定量的な効果（数値で示す）

6. スケジュール・ロードマップ
   - フェーズ別の実施計画

7. コスト・リソース
   - 予算・工数・必要リソース

8. まとめ・Next Action
   - 結論と依頼事項
```

---

## 要件シート（記入して生成フローに渡す）

```yaml
# 要件シート
目的: ""
対象読者: ""
スライド枚数: 0
出力パス: "./output/ファイル名.pptx"  # output/ ディレクトリに保存する
トーン: "フォーマル"  # フォーマル | カジュアル | 技術的
テンプレートパス: null  # null = 新規作成
言語: "ja"  # ja | en
カラースキーム: "Modern Blue"  # Modern Blue | Corporate Gray | Elegant Green | Warm Red
コンテンツ:
  - スライド1:
      タイトル: ""
      本文: ""
  - スライド2:
      タイトル: ""
      本文: ""
```

## よくある要件パターン

### パターン A: 事業・企画提案

- **目的**: 経営層・上司への承認取得
- **対象**: 非技術者、意思決定者
- **重視すること**: 課題→解決策→効果の流れ、数値による根拠
- **枚数目安**: 8〜12 枚
- **トーン**: フォーマル

### パターン B: 技術報告・レポート

- **目的**: 技術共有・社内レビュー
- **対象**: エンジニア、技術者
- **重視すること**: 正確性、図・コード・フロー図の活用
- **枚数目安**: 10〜20 枚
- **トーン**: 技術的

### パターン C: 顧客向けプレゼン

- **目的**: 受注・契約獲得
- **対象**: 顧客（意思決定者・担当者）
- **重視すること**: ベネフィットの明示、信頼性・実績の提示
- **枚数目安**: 10〜15 枚
- **トーン**: フォーマル〜カジュアル（顧客に合わせる）

### パターン D: 社内研修・勉強会

- **目的**: 知識共有・スキルアップ
- **対象**: 社内メンバー
- **重視すること**: わかりやすさ、段階的な説明
- **枚数目安**: 15〜30 枚
- **トーン**: カジュアル

## 確認が取れたら

要件シートが埋まったら `pptx-generation.md` の手順に進む。

## テンプレート・参考資料

### スライドマスター用テンプレート（`template/`）

`template/` に .pptx ファイルがある場合は、それをベースとして使用すること。
スライドマスター（フォント・カラー・レイアウト・背景）が自動的に引き継がれるため、ユーザーに別途カラースキームを確認する必要はない。

```
template/
└── *.pptx  ← ここにあれば open_presentation でベースとして使う
```

### その他の参考素材（`references/`）

ブランドロゴ・カラーガイドライン・過去の資料サンプル等は `references/` ディレクトリを確認すること。

```
references/
├── ブランドカラー・ロゴ等の画像素材
└── 過去の資料サンプル
```
